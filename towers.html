Towers of Hanoi

Uninhibited & Adventurous

Towers of Hanoi is a very peculiar puzzle game. The goal of the game is to get any number of rings from the first "starter" peg to the last "goal" peg by moving one ring at a time and without placing a larger ring on top its smaller siblings. Confused yet?

After doing a document dive on the game and playing it countless times. I found that most programs could regurgitate the fact that the game is won in the least number of moves equal to n2-1. "n" being the number of disks or rings in play. However, I did not find a single game play that could provide the exact moves that would replicate that "least number of moves" solution. So my goal was to create a recursive function that always resulted in the smallest number of moves for the given number of disks and prepended the moves to a list for users to take note of.

I also took a game-design class during undegrad so it was fun to get back to making things for the pure enjoyment of enteraining. :)
